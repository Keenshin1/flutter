Módulo 01
Variáveis e print()
O ponto de partida de qualquer programa Dart: exibir dados e guardar informações na memória.
O ponto de entrada: main()
Todo programa Dart começa pela função main(). É o ponto de entrada — quando você executa o programa, o Dart procura essa função e começa a rodar o código dentro dela.
Dart
void main() {
  // seu código vai aqui
}
void significa que a função não retorna nenhum valor. Todo programa Dart precisa de uma função main() para ser executado.
Exibindo dados com print()
A função print() exibe qualquer valor no console. É a forma mais simples de ver o resultado do seu código.
Dart
void main() {
  print('Olá, Dart!');
  print(42);
  print(3.14);
  print(true);
}
Saída no console
Olá, Dart!
42
3.14
true
Variáveis
Uma variável guarda um valor na memória. Em Dart você pode usar var (o tipo é inferido automaticamente) ou declarar o tipo de forma explícita.
Dart — com var
void main() {
  var nome   = 'Tainá';  // Dart infere: String
  var idade  = 28;      // Dart infere: int
  var altura = 1.78; // Dart infere: double
  var ativo  = true; // Dart infere: bool

  print(nome);
  print(idade);
  print(altura);
  print(ativo);
}
Dart — tipo explícito
void main() {
  String cidade = 'Curitiba';
  int    ano    = 2025;
  double pi     = 3.14159;
  bool   aberto = false;

  print(cidade);
  print(ano);
}
Regras para nomes de variáveis
Começam com letra ou _ · sem espaços · sensíveis a maiúsculas (nome ≠ Nome) · use camelCase: nomeCompleto, idadeMaxima.
Prévia: combinando variáveis em uma frase
Para incluir o valor de uma variável dentro de uma String, o Dart usa o símbolo $ antes do nome da variável. Você aprenderá tudo sobre strings no Módulo 03 — mas vale conhecer desde já:
Dart
void main() {
  var nome  = 'Ana';
  var idade = 25;

  print('Olá, $nome! Você tem $idade anos.');
  // Saída: Olá, Ana! Você tem 25 anos.
}
Use $variavel para valores simples. Use ${expressao} quando quiser fazer um cálculo ou adicionar texto grudado ao valor: 'altura: ${altura}m'.

Módulo 02
Tipos de Dados
Dart é fortemente tipado — cada variável tem um tipo que determina quais operações você pode fazer com ela.
Os tipos básicos
int
Números inteiros: 0, 42, -7
double
Números decimais: 3.14, -0.5
String
Texto: 'olá', "mundo"
bool
Verdadeiro ou falso
Dart
void main() {
  int    quantidade = 10;
  double preco      = 29.90;
  String produto    = 'Café';
  bool   disponivel = true;

  print(quantidade);
  print(preco);
  print(produto);
  print(disponivel);
}
final e const
Quando um valor não deve mudar após ser definido, use final ou const. Isso deixa o código mais seguro e legível.
Dart
void main() {
  final nome = 'Tainá';     // definido em tempo de execução, não muda
  const pi   = 3.14159; // constante em tempo de compilação

  print(nome);
  print(pi);

  // nome = 'outra'; // ERRO: final não pode ser reatribuído
}
final vs const
final é calculado uma vez em tempo de execução. const deve ser um valor completamente conhecido em tempo de compilação (números literais, strings fixas etc.).
Conversão entre tipos
Dart não converte tipos automaticamente. Use os métodos de conversão quando precisar transformar um tipo em outro.
Dart
void main() {
  // String para número
  int    a = int.parse('42');
  double b = double.parse('3.14');

  // Número para String
  String s = 100.toString();

  // int para double e vice-versa
  double d = 5.toDouble();
  int    i = 3.9.toInt();   // trunca: vira 3

  print('$a $b $s $d $i');
}
Lidando com erros de conversão: try / catch
O que acontece se você tentar converter "banana" para int? O programa lança um erro e trava. O bloco try/catch captura esse erro e permite que o programa continue funcionando normalmente.
Dart
void main() {
  try {
    int n = int.parse('banana');  // lança erro!
    print(n);
  } catch (e) {
    print('Não foi possível converter: $e');
  }

  print('O programa continua normalmente.');
}
Saída:
Não foi possível converter: FormatException: Invalid radix-10 number (at character 1)
O programa continua normalmente.
O bloco try tenta executar o código. Se ocorrer qualquer erro, o Dart pula para o bloco catch — onde e contém informações sobre o erro. O código após o try/catch continua executando normalmente. Você usará isso bastante quando lidar com dados externos (formulários, APIs, arquivos).

Módulo 03
Trabalhando com Strings
Strings são o tipo mais usado em programas reais. Dart oferece interpolação e métodos poderosos para manipulá-las.
Interpolação de strings
Em vez de concatenar strings com +, Dart permite embutir variáveis e expressões diretamente dentro de uma string usando $variavel ou ${expressão}.
Dart
void main() {
  String nome  = 'Tainá';
  int    idade = 28;

  print('Olá, $nome!');                        // Olá, Tainá!
  print('Idade: $idade anos');                 // Idade: 28 anos
  print('Daqui a 5 anos: ${idade + 5} anos');  // Daqui a 5 anos: 33 anos
  print('Nome em maiúsculas: ${nome.toUpperCase()}');
}
Use $variavel para variáveis simples e ${expressão} quando precisar calcular algo ou chamar um método dentro da string.
Métodos essenciais
Dart
void main() {
  String s = '  Dart é incrivel!  ';

  print(s.length);               // quantidade de caracteres
  print(s.trim());               // remove espaços das bordas
  print(s.toUpperCase());        // DART É INCRIVEL!
  print(s.toLowerCase());        // dart é incrivel!
  print(s.contains('Dart'));    // true
  print(s.replaceAll('!', '.')); // troca caracteres
}
Dart — split e substring
void main() {
  String frase = 'maçã,banana,uva';

  var frutas = frase.split(','); // divide em lista
  print(frutas);              // [maçã, banana, uva]

  print(frase.substring(0, 5)); // maçã,
  print(frase.startsWith('ma')); // true
  print(frase.endsWith('uva'));  // true
}
Strings multilinha
Dart
void main() {
  String texto = '''
Linha 1
Linha 2
Linha 3
''';
  print(texto);
}

Módulo 04
Operadores
Operadores são os símbolos que permitem calcular, comparar e combinar valores em Dart.
Operadores aritméticos
Dart
void main() {
  print(10 + 3);  // 13  adição
  print(10 - 3);  // 7   subtração
  print(10 * 3);  // 30  multiplicação
  print(10 / 3);  // 3.333... divisão real
  print(10 ~/ 3); // 3   divisão inteira
  print(10 % 3);  // 1   resto da divisão
}
~/ é o operador de divisão inteira do Dart — descarta a parte decimal. % retorna o resto.
Operadores de comparação
Operadores de comparação sempre retornam um bool (true ou false). Você usará esses operadores no próximo módulo para criar condições no código.
Dart
void main() {
  print(5 == 5);  // true   igual
  print(5 != 3);  // true   diferente
  print(5 >  3);  // true   maior
  print(5 <  3);  // false  menor
  print(5 >= 5);  // true   maior ou igual
  print(5 <= 4);  // false  menor ou igual
}
Operadores lógicos
Dart
void main() {
  bool a = true;
  bool b = false;

  print(a && b); // false  E (ambos precisam ser true)
  print(a || b); // true   OU (basta um ser true)
  print(!a);     // false  NÃO (inverte o valor)
}
Operadores de atribuição
Dart
void main() {
  int x = 10;
  x += 5;  // x = x + 5  → 15
  x -= 3;  // x = x - 3  → 12
  x *= 2;  // x = x * 2  → 24
  x ~/= 4; // x = x ~/ 4 → 6
  print(x); // 6

  int n = 0;
  n++;      // incrementa: n = 1
  n--;      // decrementa: n = 0
  print(n);
}

Módulo 05
if / else
Condicionais permitem que o programa tome decisões — executando caminhos diferentes dependendo se uma condição é verdadeira ou falsa.
Operadores de comparação
Para criar condições precisamos comparar valores. Toda comparação retorna um bool (true ou false).
Dart
void main() {
  print(5 == 5);  // true   igual
  print(5 != 3);  // true   diferente
  print(5 >  3);  // true   maior que
  print(5 <  3);  // false  menor que
  print(5 >= 5);  // true   maior ou igual
  print(5 <= 4);  // false  menor ou igual
}
Você estudou esses operadores com detalhes no módulo anterior. Aqui os usaremos para criar condições no código.
if / else
O if executa um bloco de código somente se a condição for true. O else é executado quando a condição é false.
Dart
void main() {
  int idade = 20;

  if (idade >= 18) {
    print('Maior de idade');
  } else {
    print('Menor de idade');
  }
}
else if — múltiplas condições
Quando há mais de dois casos possíveis, encadeie condições com else if. O Dart avalia de cima para baixo e executa apenas o primeiro bloco cuja condição for verdadeira.
Dart
void main() {
  double nota = 7.5;

  if (nota >= 9) {
    print('Excelente');
  } else if (nota >= 7) {
    print('Aprovado');
  } else if (nota >= 5) {
    print('Recuperação');
  } else {
    print('Reprovado');
  }
}
A ordem importa. Sempre verifique as condições do caso mais específico para o mais geral. No exemplo acima, se invertêssemos a ordem (verificar ≥5 antes de ≥7), a nota 7.5 seria classificada como "Recuperação" por engano.

Módulo 06
Condicionais
Você já conhece o if/else. Agora veja duas formas mais concisas de expressar decisões: o operador ternário e o switch/case.
Operador ternário
Para condições simples, o operador ternário permite escrever um if/else em uma só linha: condição ? valorSeTrue : valorSeFalse
Dart
void main() {
  int x = 15;
  String msg = x % 2 == 0 ? 'par' : 'ímpar';
  print('$x é $msg'); // 15 é ímpar
}
Use o ternário apenas para condições simples — uma linha. Se precisar de lógica mais complexa, prefira o if/else tradicional para manter a legibilidade.
switch / case
Use switch quando precisar comparar uma mesma variável com vários valores possíveis. É mais legível do que uma longa cadeia de else if nesses casos.
Dart
void main() {
  int dia = 3;

  switch (dia) {
    case 1:
      print('Segunda');
    case 2:
      print('Terça');
    case 3:
      print('Quarta');
    default:
      print('Outro dia');
  }
}
No Dart moderno (versão 3+) não é necessário break nos cases — cada case termina automaticamente.

Módulo 07
enum
Enums criam um conjunto fixo de valores nomeados — perfeitos para representar estados, categorias e opções predefinidas.
O que é um enum?
Imagine que você quer guardar o dia da semana. Poderia usar um número (1 a 7) ou uma String ("segunda"), mas ambos permitem valores inválidos como 9 ou "banana". Um enum restringe as opções exatamente ao que faz sentido.
Dart
enum DiaDaSemana {
  segunda,
  terca,
  quarta,
  quinta,
  sexta,
  sabado,
  domingo,
}

void main() {
  DiaDaSemana hoje = DiaDaSemana.sexta;
  print(hoje);           // DiaDaSemana.sexta
  print(hoje.name);      // sexta
}
Por convenção, o nome do enum usa PascalCase (DiaDaSemana) e os valores usam camelCase (segunda, terca). A propriedade .name retorna o nome do valor como String.
Enum com switch
Enums e switch funcionam muito bem juntos. Você verá isso constantemente no Flutter — por exemplo, ao verificar o estado de uma conexão ou o tipo de um botão.
Dart
enum Estacao { primavera, verao, outono, inverno }

void main() {
  Estacao atual = Estacao.inverno;

  switch (atual) {
    case Estacao.primavera:
      print('Flores brotando!');
    case Estacao.verao:
      print('Calor intenso.');
    case Estacao.outono:
      print('Folhas caindo.');
    case Estacao.inverno:
      print('Frio e casaco!');
  }
}
Saída: Frio e casaco!
No Dart 3+ o switch é exaustivo para enums: o compilador avisa se você esquecer de cobrir algum valor. Isso evita bugs silenciosos — uma das grandes vantagens de usar enums em vez de strings.
Enum com propriedades (enhanced enum)
Desde o Dart 2.17, enums podem ter propriedades e métodos. Isso é poderoso: ao invés de só nomear valores, você pode associar dados a cada um.
Dart
enum Planeta {
  mercurio(0.39),
  venus(0.72),
  terra(1.00),
  marte(1.52);

  const Planeta(this.distanciaUA);
  final double distanciaUA;
}

void main() {
  print(Planeta.marte.name);          // marte
  print(Planeta.marte.distanciaUA);   // 1.52

  for (var p in Planeta.values) {

do-while
O do-while garante que o bloco seja executado pelo menos uma vez, pois a condição é verificada após cada execução — ao contrário do while que verifica antes.

void main() {

  do {
    print('n = $n');
    n++;
  } while (n <= 3);
  // exibe: n = 1, n = 2, n = 3

  // Mesmo com condição falsa de início, executa uma vez:
  int x = 10;
  do {
    print('executou!'); // exibe mesmo com x > 5
  } while (x < 5);
}
Use do-while quando precisar que o bloco rode ao menos uma vez antes de checar a condição — por exemplo, exibir um menu e só sair quando o usuário escolher a opção correta.

Percorrendo e transformando
Alguns métodos de lista recebem uma função como argumento — você descreve o que fazer com cada item, e o método aplica isso à lista inteira. A notação (item) => expressão é uma função anônima: ela recebe o item e retorna o resultado da expressão.
Dart
void main() {
  var nums = [1, 2, 3, 4, 5];

  // forEach: percorre cada item
  nums.forEach((n) => print(n));

  // map: transforma cada item
  var dobros = nums.map((n) => n * 2).toList();
  print(dobros); // [2, 4, 6, 8, 10]

  // where: filtra itens
  var pares = nums.where((n) => n % 2 == 0).toList();
  print(pares);  // [2, 4]

  // join: une todos os itens numa String
  var texto = nums.join(', ');
  print(texto); // 1, 2, 3, 4, 5
}
join(separador) converte todos os itens da lista para String e os concatena usando o separador fornecido. Muito útil para exibir listas de forma legível ou montar textos compostos.
Funções anônimas: o segredo de map e where
Métodos como map() e where() recebem uma função como argumento — chamada de função anônima ou arrow function. Em vez de dar um nome à função, você a escreve diretamente dentro dos parênteses:
Dart
// Função nomeada — forma tradicional
bool ePar(int n) => n % 2 == 0;

// Função anônima — escrita direto como argumento
var pares = [1, 2, 3, 4].where((n) => n % 2 == 0).toList();
//                          ↑ parâmetro   ↑ corpo da função

// O padrão (n) => expressão é o mesmo de:
var pares2 = [1, 2, 3, 4].where(ePar).toList(); // mesma coisa
Leia (n) => n * 2 como: "para cada n, retorne n vezes 2". O => substitui { return ...; } quando o corpo tem apenas uma expressão.

Acesso por chave retorna um valor nullable
Quando você acessa um mapa por chave (mapa[chave]), o Dart retorna o tipo nullable (T?) — porque a chave pode não existir e o resultado seria null. Use o operador ?? para fornecer um valor padrão seguro.
Dart
void main() {
  var notas = {'Tainá': 9.0, 'Bruno': 7.5};

  // mapa[chave] retorna double? (pode ser null)
  double? n1 = notas['Tainá'];   // 9.0
  double? n2 = notas['Diego'];   // null (chave inexistente)

  print(n1); // 9.0
  print(n2); // null

  // Use ?? para garantir um valor padrão
  double notaDiego = notas['Diego'] ?? 0.0;
  print(notaDiego); // 0.0

  // containsKey evita acessar chaves inexistentes
  if (notas.containsKey('Bruno')) {
    print(notas['Bruno']); // 7.5
  }
}
Nunca use mapa[chave]! (com !) sem verificar antes se a chave existe — se não existir, o programa lança um erro em tempo de execução. Prefira ?? valorPadrão ou containsKey.

Módulo 14
Null Safety
Null safety é o sistema do Dart que impede que variáveis recebam null por acidente — eliminando uma das causas mais comuns de erros em aplicativos.
O problema do null
Em muitas linguagens, qualquer variável pode ser null sem aviso. Dart resolve isso no tipo: por padrão, uma variável não pode ser null. Para permitir null, você precisa declarar isso explicitamente com ?.
Dart
String  nome  = 'Ana';  // nunca pode ser null
String? apelido = null;  // pode ser null (note o ?)

nome    = null; // ERRO de compilação: String não aceita null
apelido = null; // OK: String? aceita null
O compilador do Dart verifica null safety antes de rodar o programa. Se um erro for possível, ele aparece no editor — não em produção.
Operador ?? — valor padrão
O operador ?? retorna o valor da esquerda se ele não for null; caso contrário, retorna o valor da direita.
Dart
String? cidade = null;

String exibir = cidade ?? 'Cidade não informada';
print(exibir); // Cidade não informada

cidade = 'São Paulo';
print(cidade ?? 'Cidade não informada'); // São Paulo
Operador ?. — acesso seguro
O operador ?. chama um método ou acessa uma propriedade somente se o objeto não for null. Se for null, o resultado inteiro é null em vez de lançar um erro.
Dart
String? texto = null;

print(texto.length);   // ERRO: texto pode ser null
print(texto?.length);  // OK: retorna null se texto for null

// Combinando ?. com ?? para garantir um valor
int tamanho = texto?.length ?? 0;
print(tamanho); // 0
Operador ! — forçar não-null
O operador ! diz ao Dart: "confie em mim, este valor não é null agora". Se você estiver errado, o programa lança um erro em tempo de execução — por isso use com cautela, somente quando tiver certeza.
Dart
String? input = obterEntrada(); // pode retornar null

// Verificando antes de usar !
if (input != null) {
  print(input.toUpperCase()); // Dart já sabe que não é null aqui
}

// Ou forçando com ! (só se tiver certeza)
print(input!.toUpperCase()); // erro em runtime se input for null
Prefira o if ao ! sempre que possível. O bloco if (x != null) é mais seguro: o Dart reconhece a verificação e trata x como não-nulo dentro do bloco automaticamente.
late — inicialização tardia
Quando você sabe que uma variável não será null, mas não pode inicializá-la na declaração (por exemplo, em classes que recebem o valor depois do construtor), use late.
Dart
class Conexao {
  late String host; // será atribuído antes do uso

  void configurar(String h) {
    host = h;
  }

  void conectar() => print('Conectando a $host');
}

void main() {
  var c = Conexao();
  c.configurar('api.exemplo.com');
  c.conectar(); // Conectando a api.exemplo.com
}

Módulo 15
async / await
Operações que levam tempo — buscar dados, ler arquivos, esperar uma resposta — são representadas por Future. O Dart resolve isso de forma elegante com async e await.
O problema: operações demoradas
Imagine um programa que busca a temperatura atual na internet. Esse processo pode levar 1, 2 ou 3 segundos. Se o programa simplesmente travasse esperando, a interface congelaria. O Dart usa Futures para representar valores que ainda não chegaram.
Dart
// Future<String> = "uma String que virá no futuro"
Future<String> buscarNome() {
  return Future.delayed(
    const Duration(seconds: 2),
    () => 'Ana',   // valor entregue após 2 segundos
  );
}

void main() {
  buscarNome().then((nome) {
    print('Nome: $nome');
  });
  print('Aguardando...'); // imprime ANTES de "Nome: Ana"
}
Saída:
Aguardando...
Nome: Ana
O método .then() recebe uma função que será chamada quando o Future completar. O código continua executando enquanto espera — por isso "Aguardando..." aparece primeiro.
async / await — código assíncrono parece síncrono
Usar .then() funciona, mas fica complicado quando há várias operações encadeadas. A sintaxe async/await permite escrever código assíncrono como se fosse linear — muito mais fácil de ler.
Dart
Future<String> buscarNome() async {
  await Future.delayed(const Duration(seconds: 2));
  return 'Ana';
}

void main() async {
  print('Buscando nome...');
  String nome = await buscarNome();  // pausa aqui e espera
  print('Nome: $nome');
  print('Concluído!');
}
Saída:
Buscando nome...
Nome: Ana
Concluído!
Regras essenciais:
· Toda função que usa await deve ser marcada com async
· Uma função async sempre retorna um Future (mesmo que você escreva void)
· await só pode ser usado dentro de funções async
Tratamento de erros com try / catch
Operações assíncronas podem falhar: a rede pode estar fora, o arquivo pode não existir. O try/catch captura esses erros sem travar o programa.
Dart
Future<int> buscarIdade(String texto) async {
  await Future.delayed(const Duration(seconds: 1));
  return int.parse(texto);  // lança erro se texto não for número
}

void main() async {
  try {
    int idade = await buscarIdade('abc');
    print('Idade: $idade');
  } catch (e) {
    print('Erro: $e');  // captura o erro de parse
  }
}
Saída: Erro: FormatException: Invalid radix-10 number (at character 1)
O bloco try contém o código que pode falhar. O bloco catch captura qualquer exceção e recebe o objeto de erro em e. Use finally para código que deve rodar sempre, com ou sem erro.
Múltiplos awaits em sequência
Dart
Future<String> buscarUsuario() async {
  await Future.delayed(const Duration(seconds: 1));
  return 'Ana';
}

Future<String> buscarCidade(String usuario) async {
  await Future.delayed(const Duration(seconds: 1));
  return 'Curitiba';
}

void main() async {
  String usuario = await buscarUsuario();
  String cidade  = await buscarCidade(usuario);
  print('$usuario mora em $cidade');
}
O await em cada linha faz o programa esperar o resultado antes de continuar — como se fossem operações síncronas normais. Isso é o grande poder do async/await: código limpo e legível mesmo para operações complexas.

Módulo 17
StatefulWidget
Widgets que reagem a interações do usuário e atualizam a tela automaticamente — o coração dos apps Flutter interativos.
Stateless vs Stateful
No módulo anterior você usou StatelessWidget — widgets que sempre exibem a mesma coisa. Um StatefulWidget possui um estado interno que pode mudar: cada vez que o estado muda, o Flutter redesenha automaticamente apenas esse widget.
Dart — estrutura mínima
class Contador extends StatefulWidget {
  const Contador({super.key});

  @override
  State<Contador> createState() => _ContadorState();
}

class _ContadorState extends State<Contador> {
  int _valor = 0;  // estado interno

  @override
  Widget build(BuildContext context) {
    return Text('$_valor');
  }
}
Um StatefulWidget sempre tem dois objetos: o widget (imutável) e o State (mutável). O Flutter cria o State via createState() e o mantém vivo entre reconstruções.
Atualizando o estado com setState()
Para mudar o estado e redesenhar o widget, envolva a modificação dentro de setState(). O Flutter ignora mudanças feitas fora dele — a tela não é atualizada.
Dart — contador completo
import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: Contador()));

class Contador extends StatefulWidget {
  const Contador({super.key});

  @override
  State<Contador> createState() => _ContadorState();
}

class _ContadorState extends State<Contador> {
  int _valor = 0;

  void _incrementar() {
    setState(() {
      _valor++;  // Flutter redesenha após esta linha
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Contador')),
      body: Center(
        child: Text(
          '$_valor',
          style: const TextStyle(fontSize: 48),
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: _incrementar,
        child: const Icon(Icons.add),
      ),
    );
  }
}
Convenção: variáveis de estado privadas começam com _ (underscore), como _valor. Isso sinaliza que pertencem à classe State e não devem ser acessadas externamente.
Múltiplas variáveis de estado
Um State pode ter quantas variáveis precisar. Todas podem ser atualizadas dentro de um único setState() — o Flutter faz apenas um redesenho.
Dart
class _PlacarState extends State<Placar> {
  int _timeA = 0;
  int _timeB = 0;

  void _gol(String time) {
    setState(() {
      if (time == 'A') _timeA++;
      else _timeB++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Row(
      mainAxisAlignment: MainAxisAlignment.center,
      children: [
        Text('$_timeA', style: const TextStyle(fontSize: 40)),
        const Text(' × ', style: TextStyle(fontSize: 40)),
        Text('$_timeB', style: const TextStyle(fontSize: 40)),
      ],
    );
  }
}
Regra de ouro: tudo que a UI precisa exibir de forma dinâmica vira variável de estado. Dados fixos (títulos, cores hardcoded) ficam direto no build().

Desafio
Crie um placar para dois times (Time A e Time B). Cada time tem seu próprio contador de gols (começa em 0). Exiba os placares lado a lado com um " × " no centro, em fonte grande. Abaixo dos placares, mostre o vencedor atual (ou "Empate"). Adicione um botão "Gol A", um botão "Gol B" e um botão "Reiniciar" que zera ambos os placares.
Solução
import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: Placar()));

class Placar extends StatefulWidget {
  const Placar({super.key});

  State<Placar> createState() => _PlacarState();

class _PlacarState extends State<Placar> {

  int _golsB = 0;

    if (_golsA > _golsB) return 'Vencedor: Time A';

    return 'Empate';

  @override
  Widget build(BuildContext context) {

    return Scaffold(
      appBar: AppBar(title: const Text('Placar')),
      body: Column(
        mainAxisAlignment: MainAxisAlignment.center,
        children: [
          Row(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              Text('$_golsA', style: placarStyle),
              const Text(' × ', style: TextStyle(fontSize: 48)),
              Text('$_golsB', style: placarStyle),
            ],
          ),
          const SizedBox(height: 8),
          Text(_vencedor, style: const TextStyle(fontSize: 20)),
          const SizedBox(height: 32),
          Row(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              ElevatedButton(
                onPressed: () => setState(() => _golsA++),
                child: const Text('Gol A'),
              ),
              const SizedBox(width: 16),
              ElevatedButton(
                onPressed: () => setState(() => _golsB++),
                child: const Text('Gol B'),
              ),
            ],
          ),
          const SizedBox(height: 12),
          TextButton(
            onPressed: () => setState(() { _golsA = 0; _golsB = 0; }),
            child: const Text('Reiniciar'),
          ),
        ],
      ),
    );
  }
}
Módulo 18
Navegação entre telas
Todo app real tem mais de uma tela. O Flutter usa uma pilha de rotas — você empurra uma nova tela por cima e volta removendo ela da pilha.
Navigator.push e Navigator.pop
O Navigator gerencia uma pilha de telas. push() coloca uma nova tela no topo; pop() remove a tela atual e volta para a anterior. O botão de voltar da AppBar chama pop() automaticamente.
Dart
import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: TelaInicial()));

class TelaInicial extends StatelessWidget {
  const TelaInicial({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Tela Inicial')),
      body: Center(
        child: ElevatedButton(
          onPressed: () {
            Navigator.push(
              context,
              MaterialPageRoute(builder: (_) => const TelaDetalhe()),
            );
          },
          child: const Text('Ir para Detalhe'),
        ),
      ),
    );
  }
}

class TelaDetalhe extends StatelessWidget {
  const TelaDetalhe({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Detalhe')),
      body: Center(
        child: ElevatedButton(
          onPressed: () => Navigator.pop(context),
          child: const Text('Voltar'),
        ),
      ),
    );
  }
}
MaterialPageRoute define a animação de transição entre telas. O parâmetro builder recebe o contexto e retorna o widget da nova tela.
Passando dados para outra tela
Para enviar dados a uma tela, basta passar argumentos pelo construtor do widget de destino — exatamente como em qualquer classe Dart.
Dart
class TelaProduto extends StatelessWidget {
  final String nome;
  final double preco;

  const TelaProduto({super.key, required this.nome, required this.preco});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text(nome)),
      body: Center(
        child: Text('R\$ $preco', style: const TextStyle(fontSize: 32)),
      ),
    );
  }
}

// Na tela que navega:
Navigator.push(
  context,
  MaterialPageRoute(
    builder: (_) => const TelaProduto(nome: 'Café', preco: 12.50),
  ),
);
Retornando dados com pop()
pop() pode devolver um valor para a tela anterior. Quem fez o push() recebe esse valor com await, pois a navegação é assíncrona.
Dart
// Tela que retorna um valor:
Navigator.pop(context, 'opcao_escolhida');

// Tela que aguarda o retorno:
final resultado = await Navigator.push<String>(
  context,
  MaterialPageRoute(builder: (_) => const TelaEscolha()),
);

if (resultado != null) {
  print('Usuário escolheu: $resultado');
}
O retorno de push() é Future<T?> — pode ser null se o usuário pressionar o botão de voltar sem chamar pop() explicitamente. Sempre verifique antes de usar.

Módulo 19
Formulários e entrada de dados
Coletar informações do usuário é essencial em qualquer app. O Flutter oferece TextField, Form e validação integrada para tornar isso seguro e organizado.
Ciclo de vida de um widget com estado
Todo StatefulWidget passa por fases durante sua existência. Conhecer essas fases é essencial para este módulo, porque o TextEditingController precisa ser criado e descartado nos momentos certos.
Dart — ciclo de vida
class _MeuState extends State<MeuWidget> {

  @override
  void initState() {
    super.initState();
    // ① chamado UMA VEZ quando o widget é inserido na tela
    //   → inicialize controllers, streams, requisições
  }

  @override
  Widget build(BuildContext context) {
    // ② chamado toda vez que setState() é chamado
    //   → reconstrói a interface com os novos dados
    return Container();
  }

  @override
  void dispose() {
    // ③ chamado UMA VEZ quando o widget é removido da tela
    //   → cancele controllers, feche streams, libere recursos
    super.dispose();
  }
}
Nunca esqueça de chamar super.initState() e super.dispose() — o Flutter precisa executar sua própria lógica nesses métodos. Omitir o super causa erros difíceis de rastrear.
TextField e TextEditingController
O TextField é o widget de entrada de texto do Flutter. Para ler o valor digitado, use um TextEditingController — ele fica vinculado ao campo e expõe o texto via .text. Crie o controller como campo da classe e sempre descarte-o no dispose() para liberar memória.
Dart
class _FormState extends State<MeuForm> {
  final _controller = TextEditingController();
  String _resultado = '';

  @override
  void dispose() {
    _controller.dispose(); // libera o controller ao destruir o widget
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Column(children: [
      TextField(
        controller: _controller,
        decoration: const InputDecoration(labelText: 'Seu nome'),
      ),
      ElevatedButton(
        onPressed: () => setState(() => _resultado = _controller.text),
        child: const Text('Confirmar'),
      ),
      Text('Olá, $_resultado!'),

O ListView simples recebe uma lista fixa de filhos em children. Para itens estruturados (ícone + título + subtítulo), use o widget ListTile — ele segue automaticamente as proporções do Material Design.

ListView.builder — listas eficientes
O ListView.builder constrói cada item sob demanda, apenas quando ele entra na área visível. É a escolha certa para qualquer lista de tamanho variável — de 10 a 10.000 itens o desempenho se mantém.
Dart
final frutas = ['Maçã', 'Banana', 'Laranja', 'Manga', 'Uva'];

ListView.builder(
  itemCount: frutas.length,
  itemBuilder: (context, index) {
    return ListTile(
      leading: CircleAvatar(child: Text('${index + 1}')),
      title: Text(frutas[index]),
      onTap: () => print('Selecionou: ${frutas[index]}'),

Combinando ListView.builder com setState(), é possível adicionar e remover itens em tempo real. A lista em memória é modificada dentro do setState() e o Flutter reconstrói apenas os itens necessários.
Dart
class _ListaState extends State<MinhaLista> {
  final _itens = <String>['Flutter', 'Dart', 'Firebase'];

  void _adicionar(String item) =>
      setState(() => _itens.add(item));

  void _remover(int index) =>
      setState(() => _itens.removeAt(index));

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: ListView.builder(
        itemCount: _itens.length,
        itemBuilder: (_, i) => ListTile(
          title: Text(_itens[i]),
          trailing: IconButton(
            icon: const Icon(Icons.delete),
            onPressed: () => _remover(i),
          ),
        ),

        child: const Icon(Icons.add),
      ),
    );
  }
}

Módulo 21
Async e Future
Operações demoradas — buscar dados de uma API, ler um arquivo — não podem travar a tela. O Dart resolve isso com Future, async e await.
Future, async e await
Um Future<T> representa um valor que estará disponível no futuro. Marque a função com async e use await para esperar o resultado sem bloquear o restante do app.
Dart
Future<String> buscarNome() async {
  // simula um atraso de rede
  await Future.delayed(const Duration(seconds: 2));
  return 'Tainá';
}

void main() async {
  print('Buscando...');
  final nome = await buscarNome(); // espera 2 segundos
  print('Nome: $nome');             // Nome: Tainá
}
await só pode ser usado dentro de funções marcadas com async. A função sempre retorna um Future — mesmo que você escreva Future<void>, o Dart garante que o retorno seja assíncrono.
FutureBuilder — conectando async à UI
O FutureBuilder é um widget que reconstrói a UI conforme o estado do Future: aguardando, concluído com dados ou concluído com erro. Ele elimina a necessidade de gerenciar manualmente flags de carregamento com setState().
Dart
Future<String> buscarMensagem() async {
  await Future.delayed(const Duration(seconds: 2));
  return 'Dados carregados!';
}

FutureBuilder<String>(
  future: buscarMensagem(),
  builder: (context, snapshot) {
    if (snapshot.connectionState == ConnectionState.waiting) {
      return const CircularProgressIndicator();
    }
    if (snapshot.hasError) {
      return Text('Erro: ${snapshot.error}');
    }
    return Text(snapshot.data!);
  },
)
Declare o Future fora do build() — nunca chame a função diretamente no parâmetro future do FutureBuilder dentro do build(). Isso recriaria o Future a cada reconstrução, reiniciando o carregamento infinitamente.
Tratamento de erros com try/catch
Operações assíncronas podem falhar — sem internet, servidor fora do ar, dados inválidos. Use try/catch dentro de funções async para capturar exceções e exibir feedback adequado ao usuário.
Dart
Future<String> buscarComFalha() async {
  await Future.delayed(const Duration(seconds: 1));
  throw 'Servidor indisponível'; // simula falha
}

Future<void> carregar() async {
  try {
    final dados = await buscarComFalha();
    print(dados);
  } catch (e) {
    print('Erro capturado: $e'); // Erro capturado: Servidor indisponível
  }
}
Boas práticas: sempre trate erros em chamadas assíncronas. Exiba um estado de erro claro na UI — uma mensagem com botão "Tentar novamente" é muito melhor do que uma tela em branco ou travada.

Módulo 22
Streams — dados em tempo real
Enquanto um Future entrega um único valor no futuro, um Stream entrega vários valores ao longo do tempo — perfeito para notificações, sensores, chats e qualquer dado que muda continuamente.
Stream vs Future
Um Future<T> resolve uma vez — com sucesso ou erro. Um Stream<T> é um canal que pode emitir zero ou mais valores ao longo do tempo e fecha quando não há mais dados. Você percorre um stream com await for, da mesma forma que usa await com um Future.
Dart
// Future — um único valor
Future<int> umValor() async {
  await Future.delayed(Duration(seconds: 1));
  return 42;
}

// Stream — vários valores ao longo do tempo
Stream<int> contagem() async* {
  for (var i = 1; i <= 5; i++) {
    await Future.delayed(Duration(seconds: 1));
    yield i;  // emite um valor e continua
  }
}

void main() async {
  await for (final n in contagem()) {
    print('Recebido: $n');
  }
  print('Stream encerrado!');
}
Saída: Recebido: 1 → Recebido: 2 → ... → Recebido: 5 → Stream encerrado!
async* e yield andam juntos: async* declara que a função é um gerador assíncrono, e yield emite um valor sem encerrar a função — ela fica "pausada" até emitir o próximo.
StreamController — controle total
Quando você precisa emitir eventos de forma imperativa (por exemplo, ao receber dados de um WebSocket ou ao clicar em botões), use o StreamController<T>. Ele expõe um sink para adicionar dados e um stream para ouvir.
Dart
import 'dart:async';

void main() async {
  final controller = StreamController<String>();

  // ouvinte: processa cada evento
  controller.stream.listen((msg) {
    print('Mensagem: $msg');
  });

  // produtores: adicionam dados ao canal
  controller.sink.add('Olá');
  controller.sink.add('Flutter');
  controller.sink.add('Streams!');

  await Future.delayed(Duration(milliseconds: 100));
  await controller.close(); // sempre feche quando terminar
}
Fechar o StreamController com close() é obrigatório para evitar vazamentos de memória. Em widgets Flutter, faça isso no dispose().
StreamBuilder — conectando ao widget tree
O StreamBuilder é o equivalente do FutureBuilder para streams. Ele reconstrói a UI a cada novo evento, recebendo um AsyncSnapshot com o estado atual: aguardando, com dado, ou com erro.
Dart
import 'package:flutter/material.dart';

// stream que emite o tempo decorrido a cada segundo
Stream<int> relogio() async* {
  var seg = 0;
  while (true) {
    await Future.delayed(const Duration(seconds: 1));
    yield ++seg;
  }
}

class Cronometro extends StatelessWidget {
  const Cronometro({super.key});

  @override
  Widget build(BuildContext context) {
    return StreamBuilder<int>(
      stream: relogio(),
      builder: (context, snapshot) {
        if (!snapshot.hasData) {
          return const Text('Iniciando...');
        }
        return Text(
          '${snapshot.data}s',
          style: const TextStyle(fontSize: 48),
        );
      },
    );
  }
}
Note que Cronometro é um StatelessWidget — o StreamBuilder gerencia toda a atualização de estado internamente. Não é necessário chamar setState().
Transformando streams: map e where
Streams possuem métodos funcionais semelhantes aos de List: map() transforma cada evento, where() filtra eventos que não atendem a um critério, e listen() registra um ouvinte sem usar await for. Você pode encadear esses métodos assim como em coleções.
Dart
import 'dart:async';

Stream<int> numeros() async* {
  for (var i = 1; i <= 10; i++) {
    await Future.delayed(Duration(milliseconds: 200));
    yield i;
  }
}

void main() async {
  await numeros()
      .where((n) => n.isEven)          // só números pares
      .map((n) => 'Par: $n')            // transforma em String
      .forEach(print);                   // Par: 2, Par: 4, ..., Par: 10
}
Saída: Par: 2 → Par: 4 → Par: 6 → Par: 8 → Par: 10

Contador regressivo
async* e yield
Básico
Crie uma função regressivo(int inicio) que retorna um Stream<int> emitindo os números de inicio até 1 com intervalo de 500 ms entre cada um. No main(), use await for para imprimir cada número e, ao final, imprima "Lançar!".

import 'dart:async';

Stream<int> regressivo(int inicio) async* {
  for (var i = inicio; i >= 1; i--) {
    await Future.delayed(const Duration(milliseconds: 500));
    yield i;
  }
}

void main() async {
  await for (final n in regressivo(5)) {
    print(n);
  }
  print('Lançar!');
}

Filtro de temperatura
StreamController + where + map
Médio
Crie um StreamController<double> chamado sensor. Adicione ao sink as temperaturas: 18.5, 23.0, 37.2, 25.1, 40.8, 22.0. Ouça o stream com listen(), filtrando apenas temperaturas acima de 35°C e imprimindo no formato "ALERTA: 37.2°C". Feche o controller ao final.
Solução
import 'dart:async';

void main() async {
  final sensor = StreamController<double>();

  sensor.stream
      .where((t) => t > 35)
      .map((t) => 'ALERTA: ${t.toStringAsFixed(1)}°C')
      .listen(print);

  for (final temp in [18.5, 23.0, 37.2, 25.1, 40.8, 22.0]) {
    sensor.sink.add(temp);
  }

  await sensor.close();
}
03
Cronômetro interativo
StreamBuilder + StatefulWidget + StreamController
Desafio
Crie um app Flutter com um StatefulWidget chamado Cronometro. Ele deve ter um StreamController<int> e dois botões: Iniciar e Resetar. Ao iniciar, use um Timer.periodic (de dart:async) para emitir um inteiro crescente no sink a cada segundo. O StreamBuilder exibe o tempo no formato "00:07" (minutos:segundos). Ao resetar, cancele o timer, zere o contador e emita 0 no sink. Feche o controller no dispose().
Solução
import 'dart:async';
import 'package:flutter/material.dart';

void main() => runApp(const MaterialApp(home: Scaffold(body: Center(child: Cronometro()))));

class Cronometro extends StatefulWidget {
  const Cronometro({super.key});
  @override
  State<Cronometro> createState() => _CronometroState();
}

class _CronometroState extends State<Cronometro> {
  final _ctrl = StreamController<int>();
  Timer? _timer;
  var _segundos = 0;
  var _rodando = false;

  void _iniciar() {
    if (_rodando) return;
    setState(() => _rodando = true);
    _timer = Timer.periodic(const Duration(seconds: 1), (_) {
      _ctrl.sink.add(++_segundos);
    });
  }

  void _resetar() {
    _timer?.cancel();
    setState(() { _rodando = false; _segundos = 0; });
    _ctrl.sink.add(0);
  }

  String _formatar(int s) {
    final m = s ~/ 60;
    return '${m.toString().padLeft(2,'0')}:${(s % 60).toString().padLeft(2,'0')}';
  }

  @override
  void dispose() {
    _timer?.cancel();
    _ctrl.close();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Column(mainAxisSize: MainAxisSize.min, children: [
      StreamBuilder<int>(
        stream: _ctrl.stream,
        initialData: 0,
        builder: (_, snap) => Text(
          _formatar(snap.data!),
          style: const TextStyle(fontSize: 64, fontWeight: FontWeight.bold),
        ),
      ),
      const SizedBox(height: 24),
      Row(mainAxisAlignment: MainAxisAlignment.center, children: [
        ElevatedButton(onPressed: _iniciar, child: const Text('Iniciar')),
        const SizedBox(width: 16),
        OutlinedButton(onPressed: _resetar, child: const Text('Resetar')),
      ]),
    ]);
  }
}
