Módulo 01
Introdução ao Flutter
Conheça o framework do Google para criar apps bonitos em múltiplas plataformas com um único código.
O que é Flutter?
Flutter é um SDK de código aberto criado pelo Google que permite criar apps para Android, iOS, Web e Desktop usando um único código-base, escrito na linguagem Dart.
Vantagem principalEscreva uma vez, rode em qualquer lugar — o mesmo código funciona em celular, navegador e computador sem reescrever nada.
Estrutura mínima de um app
Todo app Flutter começa com a função main(), que chama runApp() passando o widget raiz.
Dart
import 'package:flutter/material.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return const MaterialApp(
      home: Scaffold(
        body: Center(
          child: Text('Olá Flutter!'),
        ),
      ),
    );
  }
}
Fluxo de execuçãomain() → runApp() → widget raiz → build() → tela renderizada
MaterialApp e Scaffold
MaterialApp configura o app todo (tema, rotas, título). Scaffold oferece a estrutura visual básica de uma tela: AppBar, body, FloatingActionButton etc.

02

Juntando tudo que aprendeu
Crie um app Flutter completo que exibe "Meu Primeiro App" centralizado na tela. Use MaterialApp, Scaffold, Center e Text.
Solução
import 'package:flutter/material.dart';

void main() => runApp(const MeuApp());

class MeuApp extends StatelessWidget {

Módulo 02
Tudo é Widget
Em Flutter, cada elemento visual é um widget — do texto mais simples à tela inteira.
O conceito de Widget
Em Flutter, tudo é widget. Textos, botões, imagens, espaçamentos — cada peça da interface é um widget. Eles se combinam como peças de Lego para formar a tela completa.
StatelessWidget
Sem estado — não muda após renderizado
StatefulWidget
Com estado — pode se atualizar dinamicamente
StatelessWidget
Um StatelessWidget é um widget estático. Você sobrescreve o método build() para dizer o que ele deve desenhar na tela.
Dart
class MeuWidget extends StatelessWidget {
  const MeuWidget({super.key});

  @override
  Widget build(BuildContext context) {
    return const Text('Olá, Flutter!');
  }
}

child vs childrenA propriedade child recebe um único widget filho. Para múltiplos filhos, use children (uma lista) com Column ou Row.

Módulo 03
Layout com Column e Row
Organize widgets vertical e horizontalmente para construir qualquer tipo de tela.
Scaffold e AppBar
O Scaffold é o esqueleto de uma tela. Ele aceita uma AppBar no topo e um body onde vai o conteúdo principal.
Dart
Scaffold(
  appBar: AppBar(
    title: const Text('Meu App'),
    backgroundColor: Colors.blue,
  ),
  body: const Center(
    child: Text('Conteúdo aqui'),
  ),
)
Column e Row
Column organiza widgets um abaixo do outro (vertical). Row organiza widgets lado a lado (horizontal). Ambos aceitam uma lista em children.
Dart — Column
Column(
  mainAxisAlignment: MainAxisAlignment.center,
  crossAxisAlignment: CrossAxisAlignment.center,
  children: const [
    Text('Linha 1'),
    Text('Linha 2'),
    Text('Linha 3'),
  ],
)
Dart — Row
Row(
  mainAxisAlignment: MainAxisAlignment.spaceEvenly,
  children: const [
    Icon(Icons.home),
    Icon(Icons.search),
    Icon(Icons.person),
  ],
)
mainAxisAlignmentstart · end · center · spaceBetween · spaceAround · spaceEvenly
Espaçamento e agrupamento
Dart
SizedBox(height: 16)         // espaço fixo vertical

Padding(
  padding: EdgeInsets.all(16),
  child: const Text('Com padding'),

Center(child: const Text('Centralizado'))

Expanded(child: Container(color: Colors.blue)) // ocupa espaço restante
Exercícios de fixação
01
Tela com AppBar
Scaffold completo com Column
Crie um Scaffold com AppBar de título "Minha Tela" e um body com Column centralizada contendo três Text: "Item 1", "Item 2" e "Item 3".
Solução
Scaffold(
  appBar: AppBar(title: const Text('Minha Tela')),
  body: const Column(
    mainAxisAlignment: MainAxisAlignment.center,
    children: [
      Text('Item 1'),
      Text('Item 2'),
      Text('Item 3'),
    ],
  ),
)
