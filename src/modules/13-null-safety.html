Módulo 12
Null Safety
Null safety é o sistema do Dart que impede que variáveis recebam null por acidente — eliminando uma das causas mais comuns de erros em aplicativos.
O problema do null
Em muitas linguagens, qualquer variável pode ser null sem aviso. Dart resolve isso no tipo: por padrão, uma variável não pode ser null. Para permitir null, você precisa declarar isso explicitamente com ?.
Dart
String  nome  = 'Ana';  // nunca pode ser null
String? apelido = null;  // pode ser null (note o ?)

nome    = null; // ERRO de compilação: String não aceita null
apelido = null; // OK: String? aceita null
O compilador do Dart verifica null safety antes de rodar o programa. Se um erro for possível, ele aparece no editor — não em produção.
Operador ?? — valor padrão
O operador ?? retorna o valor da esquerda se ele não for null; caso contrário, retorna o valor da direita.
Dart
String? cidade = null;

String exibir = cidade ?? 'Cidade não informada';
print(exibir); // Cidade não informada

cidade = 'São Paulo';
print(cidade ?? 'Cidade não informada'); // São Paulo
Operador ?. — acesso seguro
O operador ?. chama um método ou acessa uma propriedade somente se o objeto não for null. Se for null, o resultado inteiro é null em vez de lançar um erro.
Dart
String? texto = null;

print(texto.length);   // ERRO: texto pode ser null
print(texto?.length);  // OK: retorna null se texto for null

// Combinando ?. com ?? para garantir um valor
int tamanho = texto?.length ?? 0;
print(tamanho); // 0
Operador ! — forçar não-null
O operador ! diz ao Dart: "confie em mim, este valor não é null agora". Se você estiver errado, o programa lança um erro em tempo de execução — por isso use com cautela, somente quando tiver certeza.
Dart
String? input = obterEntrada(); // pode retornar null

// Verificando antes de usar !
if (input != null) {
  print(input.toUpperCase()); // Dart já sabe que não é null aqui
}

// Ou forçando com ! (só se tiver certeza)
print(input!.toUpperCase()); // erro em runtime se input for null
Prefira o if ao ! sempre que possível. O bloco if (x != null) é mais seguro: o Dart reconhece a verificação e trata x como não-nulo dentro do bloco automaticamente.
late — inicialização tardia
Quando você sabe que uma variável não será null, mas não pode inicializá-la na declaração (por exemplo, em classes que recebem o valor depois do construtor), use late.
Dart
class Conexao {
  late String host; // será atribuído antes do uso

  void configurar(String h) {
    host = h;
  }

  void conectar() => print('Conectando a $host');
}

void main() {
  var c = Conexao();
  c.configurar('api.exemplo.com');
  c.conectar(); // Conectando a api.exemplo.com
}
Conexão com Flutter
No Flutter você verá late frequentemente em State para inicializar controladores e animações no initState(), e String? em campos de formulário que ainda não foram preenchidos pelo usuário.
Exercícios de fixação
01
Perfil opcional
Declaração e operador ??
Básico
Crie as variáveis String nome = 'Lucas', String? bio (sem valor) e int? idade (sem valor). Use ?? para exibir "Sem bio" caso bio seja null e 0 caso idade seja null.
Solução
void main() {
  String  nome  = 'Lucas';
  String? bio;
  int?    idade;

  print(nome);
  print(bio   ?? 'Sem bio'); // Sem bio
  print(idade ?? 0);         // 0
}
02
Validador de cadastro
Operadores ?. e ?? com classes
Médio
Crie a classe Usuario com String nome e String? email. Escreva a função exibirInfo(Usuario? u) que: (1) se u for null, imprime "Usuário não encontrado"; (2) caso contrário, exibe o nome e o email (usando ?. e ?? para tratar email null). Teste com um usuário completo, um sem email e passando null.
Solução
class Usuario {
  String  nome;
  String? email;
  Usuario(this.nome, {this.email});
}

void exibirInfo(Usuario? u) {
  if (u == null) {
    print('Usuário não encontrado');
    return;
  }
  print('Nome: ${u.nome}');
  print('Email: ${u.email ?? "não informado"}');
}

void main() {
  exibirInfo(Usuario('Ana', email: 'ana@email.com'));
  print('---');
  exibirInfo(Usuario('Bruno'));
  print('---');
  exibirInfo(null);
}
03
Inventário com itens opcionais
late · ?? · ?. · List nullable
Desafio
Crie a classe Produto com String nome, double preco e String? descricao. Crie a classe Inventario com uma propriedade late List<Produto> itens e o método carregar() que atribui uma lista de 4 produtos (alguns sem descrição) a itens. Implemente exibirCatalogo() que percorre os itens e exibe o nome, preço e descrição (ou "Sem descrição" se null). Calcule e exiba o total do inventário.
Solução
class Produto {
  String  nome;
  double  preco;
  String? descricao;
  Produto(this.nome, this.preco, {this.descricao});
}

class Inventario {
  late List<Produto> itens;

  void carregar() {
    itens = [
      Produto('Teclado',  250.0, descricao: 'Mecânico RGB'),
      Produto('Mouse',    120.0),
      Produto('Monitor',  1200.0, descricao: '27" Full HD'),
      Produto('Mousepad', 45.0),
    ];
  }

  void exibirCatalogo() {
    double total = 0;
    for (var p in itens) {
      print('${p.nome} — R\$ ${p.preco}');
      print('  ${p.descricao ?? "Sem descrição"}');
      total += p.preco;
    }
    print('Total: R\$ $total');
  }
}

void main() {
  var inv = Inventario();
  inv.carregar();
  inv.exibirCatalogo();
}
